Riding the Data Pipeline

Master's of Data Science Capstone Project

Overview

This project delivers an end-to-end data pipeline for analyzing public transportation ridership and fare data. The system ingests, processes, and visualizes daily rider behavior and fare usage trends using Airflow, PostgreSQL, and Grafana. Our goal was to support data-driven decisions around fare policy, rider equity, and system performance.

Skills & Tools Used

  • Python (pandas, psycopg2)
  • SQL (PostgreSQL)
  • Apache Airflow (DAG scheduling, timezone handling)
  • Grafana (dashboarding, time-series plots)
  • Cloud storage (S3 buckets)
  • Team collaboration with GitHub and Slack

Process

  • Data Ingestion: Created Airflow DAGs to scrape and export rider and fare data from APIs into S3 as JSON.
  • Data Organization: Designed and maintained dimension and fact tables in a PostgreSQL warehouse.
  • Data Loading: Used Airflow to transform and insert data from S3 into structured warehouse tables with timestamp handling.
  • Dashboarding: Built Grafana dashboards to track daily ridership, fare type usage, and performance over time.
  • Team Workflow: Iterated with teammates using GitHub issues, milestone reviews, and feedback-based revisions.

Key Outcomes

  • Automated daily ingestion and transformation of ridership and fare data using custom Airflow DAGs
  • Created robust fact and dimension tables that support flexible filtering and analysis
  • Visualized key KPIs in Grafana, including peak ridership times and equity breakdowns by fare type
  • Handled complex timezone issues and ensured timestamp accuracy across systems
  • Produced clean, well-documented code and reproducible workflows
